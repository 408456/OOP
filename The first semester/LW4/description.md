## Задание
Разработайте систему для управления клиентами и их счетами в финансовом приложении.

Система должна:

1. Поддерживать создание и управление клиентами, включая добавление различных типов счетов (например, текущий, депозитный, инвестиционный).
2. Обеспечивать возможность перевода средств между счетами, начисления процентов и ведения учета операций.
3. Включать функции для генерации отчетов по состоянию счета, истории операций и общей статистики по клиентам.
4. Поддерживать разные валюты (как минимум, рубль, юань, доллар) и получать актуальный курс из внешней системы. Например, из API ЦБ РФ.   
5. Логику необходимо покрыть unit тестами
6. За реализацию интеграционных тестов будут начислены дополнительные баллы 

**Вы работаете над системой для банка, который стремиться упростить управление финансовыми операциями. Система должна быть надежной.**

Цель ЛР заключается в применении принципов ООП и паттернов проектирования для создания безопасной, масштабируемой и легко поддерживаемой системы управления клиентами и финансовыми данными.

## Выполнение задание

### Логика
Клиент создает счета (текущий, депозитный или инвестиционный).
Счета могут выполнять операции, такие как пополнение, снятие и переводы, при этом каждая операция сохраняется в историю транзакций.
При переводах между счетами или конвертации валют используется TransferService, который проверяет достаточность средств и выполняет обмен валют по текущему курсу.
Для депозитных и инвестиционных счетов будет рассчитан процент, используя InterestCalculator, который учитывает процентные ставки, определяемые через InterestRate.
Генерация отчетов позволяет отслеживать состояние счетов, истории операций и статистику по каждому клиенту.

### Классы 

Client(имя, контакты и список счетов)

Account(баланс, валюта, история операций)(пополнение, снятие, перевдоды)

CheckingAccount, DepositAccount, InvestmentAccount - наследникик Account

Transaction (сумма, дата, тип операции, счета)(перевод, начисление процентов)

TransferService(проверка достаточности средств и валютного обмена)

Currency (код, название, курс)

CurrencyConverter (конвертатор)

Report (отчеты  по счетам и историям операций)

InterestRate(информация о процентных ставках) 

InterestCalculator (начисленные проценты по типам счетов)
